# Step2: 포트폴리오 구성 - 개발 계획

## 1. 현재 상태 요약
- **핵심 기능**: 계좌형 모델(v2), 계좌별 월 납입 비중 및 자산군 목표 비중 조절, 자동 현금 처리.
- **시각화**: 종합 도넛(월 투자 가능 금액 기준) 및 계좌별 미니 도넛 탭 제공.
- **데이터 연동**: Step1의 브리지 데이터(월 투자여력 등) 수동 가져오기, v1 레거시 자동 마이그레이션.

## 2. 다음 MVP+ 항목
- **입력 단순화**: 초기 투자/저축 기존 보유금액을 계좌별 보유금액 방식으로 전환 (TODO 보류 항목 연계).
- **저장/복원**: Step1과 동등한 수준의 URL Hash 공유 및 JSON 내보내기/불러오기 기능 추가.
- **모바일 UX**: 모바일 화면에서 계좌 및 자산군 순서 변경(Drag & Drop 또는 상하 버튼) 지원.

## 3. 스키마/호환성 리스크 및 마이그레이션 원칙
- **리스크**: 보유 자산(Actual)과 목표 자산(Target) 모델 분리 시 `step2Portfolios` 스키마 복잡도 증가 및 하위 호환성 충돌 위험.
- **원칙**: 이전 버전(v1, v2) 로드 시 데이터 유실 없이 v3(보유 자산 포함 모델)로 런타임 자동 변환 후 저장을 유도.

## 4. 우선순위 및 수용 기준(AC)
- **[High] Step2 URL Hash 공유 & JSON 내/보내기 (feat)**: IDB 없이 Hash/JSON으로 포트폴리오 상태 복원 가능. (예상 난이도: 하)
- **[High] 계좌별 기존 보유금액 입력 체계 전환 (feat)**: Step1 연계 시 기존 금액을 계좌에 매핑하여 실제 보유량과 목표 비중 갭 분석 기반 마련. (예상 난이도: 중)
- **[Med] 계좌/자산군 순서 변경 기능 (UX)**: 버튼 또는 D&D로 리스트 순서를 직관적으로 변경 가능. (예상 난이도: 중)
- **[Low] 목표 vs 현재 비중 갭 차트 (UX)**: 보유금액 입력 시 목표 비중과의 차이를 막대/도넛으로 표시. (예상 난이도: 중)
