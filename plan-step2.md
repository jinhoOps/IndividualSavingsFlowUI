# Step2: 포트폴리오 구성 - 개발 계획

기준 시점: `2026-03-26`, 기준 커밋: `eae9755`

## 1) 현재 상태 요약
- 모델: `modelVersion:2` 계좌형 구조(`totalMonthlyInvestCapacity`, `accounts[]`, `notes`, `updatedAt`)가 기본 저장 포맷.
- 편집: 계좌 비중 + 계좌 내 자산군 비중 편집, 계좌/자산군 CRUD, 합계 검증(저장 차단) 동작.
- 시각화: `종합 도넛`/`계좌별 도넛` 탭 분리와 퍼센트 정수 표기, 금액 패널 렌더 안정화 반영.
- 모바일: `mobileAccountSelect` 기반 단일 계좌 집중 편집으로 스크롤 길이 축소.
- 연동: Step1 브리지 데이터 수동 가져오기 보강 (hub 조회 실패 시 step1Snapshots → isf-rebuild-v1 로컬 포맷 순으로 자동 fallback 복원).
- 호환: v1(`targetAllocations`) 로드 시 v2 `통합계좌`로 런타임 변환 후 저장 가능.

## 2) 다음 MVP+ 범위
### 입력 단순화
- `S2-MVP1` 보유금액(Actual) 입력 도입: 목표 비중(Target)과 분리된 현재 보유량 입력 지원.
- `S2-MVP2` 빠른 입력 프리셋: 자산군 템플릿/복사/계좌 복제 기능으로 초기 세팅 시간을 단축.

### 모바일 UX
- `S2-MVP3` 계좌/자산군 순서 변경: 상하 이동 버튼 우선(드래그는 후순위)으로 조작 복잡도 최소화.
- `S2-MVP4` 입력 에러 즉시 피드백: 비중 합계 오류, 빈 이름, 0 이하 값에 대한 인라인 검증 강화.

### 시각화 고도화
- `S2-MVP5` 목표 대비 현재 갭 시각화: 계좌별/자산군별 `Target vs Actual` 차이를 바/도넛으로 제공.
- `S2-MVP6` 종합 도넛 drill-down: 종합 탭에서 계좌별 기여도와 현금 발생 원인(비중 미할당)을 즉시 확인.

### 저장/복원
- `S2-MVP7` Step2 공유/복원 경로 신설: URL hash(JSON 압축) + JSON 내보내기/불러오기.
- `S2-MVP8` 저장 충돌 완화: 최근 저장본/브리지 반영본 간 충돌 안내 및 선택 복원 UX.

## 3) 스키마/호환성 리스크와 마이그레이션 원칙
### 주요 리스크
- `S2-R1` v2에 Actual 필드 추가 시 저장 페이로드 급증으로 로드/저장 성능 저하 위험.
- `S2-R2` 브리지 payload는 Step1 기준 최소 필드만 전달되므로 Step2 확장 필드와 의미 불일치 가능.
- `S2-R3` 공유(hash/json)와 IDB 저장 포맷이 분기되면 동일 포트폴리오의 역직렬화 결과가 달라질 위험.

### 마이그레이션 원칙
1. 비파괴 우선: 로드 시 원본 객체 보존 후 새 draft로 변환, 실패 시 원본 유지 + 경고 메시지.
2. 단방향 업그레이드: `v1 -> v2 -> v3` 체인을 명시하고, 최신 저장은 항상 최신 버전으로만 수행.
3. 기본값 명시: 신규 필드(`actualHoldings`, `rebalanceMeta` 등)는 null이 아닌 안전 기본값으로 채움.
4. 가시적 버전 표기: 저장/불러오기 UI에 모델 버전 표시, 마이그레이션 발생 시 토스트/로그 남김.
5. 브리지 격리: Step1 브리지 데이터는 `bridgeContext`로만 병합하고 사용자 편집 데이터 overwrite 금지.
