# Strategy: Resolving the Financial Reality Gap

사용자가 느끼는 시뮬레이션의 괴리감은 고정된 패턴(Mock Data)과 현실의 비선형적 변동성 사이의 불일치에서 발생합니다. 이를 해소하기 위한 단계적 로드맵을 제안합니다.

## 1. 데이터의 현실화 (Data Realism)
*   **현재 문제**: `gold.json` 등 현재 데이터가 비정상적으로 우상향(3년 160%+)하여 KPI 수치가 비현실적으로 높게 나옴.
*   **해결 방안**:
    *   **Phase A (정적 실제 데이터)**: 2000년~현재까지의 QQQ, SPY, Gold, KOSPI 200의 실제 종가 데이터를 Yahoofinance 등에서 추출하여 JSON 파일로 교체. (수동 갱신 방식이지만 즉각적인 괴리감 해소 가능)
    *   **Phase B (데이터 브릿지)**: 로컬 JSON 파일 대신 외부 무료 API(Alpha Vantage, Yahoo Finance 등)를 호출하여 최신 데이터를 동적으로 로드하는 서비스 레이어 구축.

## 2. 엔진의 고도화 (Engine Sophistication)
*   **현재 문제**: 레버리지나 배당 재투자가 단순히 숫자로만 더해져 변동성 드래그(Volatility Drag) 등의 현실적 손실이 과소평가될 수 있음.
*   **해결 방안**:
    *   **변동성 드래그 구현**: 레버리지 배수 적용 시 단순 수익률 곱하기가 아닌, 일 단위 복리 계산을 통해 횡보장에서의 가치 하락(Drag)을 정밀하게 재현.
    *   **환율 변동성 반영**: 고정 환율(1,450원) 대신 실제 과거 환율(USD/KRW) 시계열 데이터를 사용하여 해외 자산 평가 시 환차익/환차손을 시뮬레이션에 포함.

## 3. 사용자 경험의 투명성 (Transparency)
*   **현재 문제**: 계산 근거가 블랙박스 상태여서 수치가 높게 나오면 의구심이 생김.
*   **해결 방안**:
    *   **로그 뷰어 추가**: 특정 시점(예: 2024년 5월)에 왜 자산이 그렇게 변했는지, "금 가격 15% 상승 + 환율 2% 상승"과 같은 상세 변동 사유를 툴팁이나 로그로 제공.
    *   **현실적 경고**: "이 수익률은 과거 특정 기간의 데이터에 기반하며, 미래를 보장하지 않습니다"와 같은 금융 면책 조항 및 백테스트의 한계점 명시.

## 🎯 우선순위 권고
1.  **High**: `public/data/indices/*.json` 파일을 실제 과거 시계열 데이터로 교체. (가장 적은 노력으로 가장 큰 체감 효과)
2.  **Medium**: 과거 환율(USD/KRW) 시계열 데이터 도입.
3.  **Low**: 실시간 API 연동 브릿지 개발.
