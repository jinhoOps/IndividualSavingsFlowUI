# [ARCHIVED] UI/UX Overhaul v0.4 (사용자 중심 적응형 개편)

> [!NOTE]
> 본 문서는 v0.4 대규모 UI/UX 개편 당시의 기록입니다. 현재의 최신 디자인 표준 및 아키텍처는 **[[Architecture_Reference]]** 및 **[[UI_Standards_Reference]]**를 참조하십시오.

본 문서는 최초 사용자와 기존 사용자 간의 인지 부하 차이를 해결하고, 파편화된 데이터 관리 기능을 통합하여 사용자 경험을 고도화한 v0.4 개편 사항을 기록합니다.

## 1. 스마트 적응형 엔트리 (Smart Adaptive Entry)
- **문제**: 기존 사용자가 매번 입력 폼을 먼저 보게 되어 결과 확인까지 불필요한 단계를 거침.
- **해결**: `localStorage` 또는 `IndexedDB`에 데이터가 존재하는 '기존 사용자' 감지 시, 앱 로드 직후 `is-dashboard-mode`를 활성화함.
- **효과**: 결과물(차트, 요약 카드)이 최상단에 배치되고 입력 패널은 자동으로 접혀, 사용자가 즉시 자산 흐름을 파악할 수 있음.

## 2. 데이터 관리 허브 통합 (Consolidated Data Hub)
- **문제**: 백업, 공유, 내보내기, 포트폴리오 저장 등 데이터 관련 기능이 여러 패널에 흩어져 있어 인지 부하가 높음.
- **해결**: `AppHeader`의 설정(톱니바퀴) 아이콘을 통해 열리는 `DataHubModal`로 모든 기능을 일원화함.
  - Step 1: 로컬 백업 및 파일 관리 중심.
  - Step 2: 저장된 포트폴리오 목록 및 백업 관리 중심 (탭 시스템 적용).
- **효과**: 메인 UI의 노이즈를 제거하고 '데이터 관리'라는 맥락 안에서 모든 작업을 수행할 수 있게 함.

## 3. 저밀도 피드백 시스템 (Silent Status Indicator)
- **문제**: 자동 저장 시마다 팝업되는 토스트 알림이 사용자의 집중을 방해하고 시스템이 불안정하다는 인상을 줌.
- **해결**: 헤더 내부에 작은 텍스트와 도트 형태의 '상태 인디케이터'를 배치함 (`updateStatus('saving'|'success'|'error')`).
- **효과**: 사용자 주의를 뺏지 않으면서도 시스템이 안전하게 동작 중임을 신뢰할 수 있게 함.

## 4. 스마트 브리지 배너 (Smart Bridge Banner)
- **문제**: Step 1 데이터 업데이트 시 강제로 팝업이 뜨거나 구석의 패널을 직접 찾아야 함.
- **해결**: 상단에 플로팅되는 비침습적 배너를 통해 새로운 데이터 감지 사실을 알리고 사용자가 원하는 시점에 적용하도록 함.
