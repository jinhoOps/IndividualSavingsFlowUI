- 개발 시작 전에 항상 `UPDATE.md`를 먼저 확인한다.

- 버전은 당분간 `0.x`(pre-1.0)로 관리한다.
- 버전 증분 규칙: `feat => minor up (0.x.0)`, `fix/chore/docs/refactor => patch up (0.x.y)`.
- 커밋 태그 규칙(기본 고정): `feat:` / `fix:` / `docs:` / `chore:`만 사용한다.
