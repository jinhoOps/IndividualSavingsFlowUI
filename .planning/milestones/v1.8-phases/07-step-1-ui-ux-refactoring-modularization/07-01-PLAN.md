<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported/share/backup/hash data -> Step 1 state | Untrusted or stale local/share data enters `sanitizeInputs` and persistence/render paths. |
| Step 1 state -> HTML templates | User/imported account, transfer, group, and item names cross into DOM rendering. |
| browser viewport/resize -> SVG renderers | Untrusted environment dimensions can trigger repeated layout/redraw work. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | `external-input-guard.js`, `persistence-controller.js` | mitigate | Normalize every external Step 1 input with `sanitizeInputs` before state commit or bridge persistence. |
| T-07-02 | Information Disclosure | `list-renderer.js`, `ui-controller.js` | mitigate | Escape or DOM-build all user/imported strings before they reach HTML, including attribute values. |
| T-07-03 | Denial of Service | `sankey-renderer.js`, `network-map-renderer.js` | mitigate | Bound resize/tab redraws with existing debounce and renderer-local early returns. |
| T-07-SC | Tampering | npm installs | accept | No package-manager install is planned or allowed for this phase. |
</threat_model>
