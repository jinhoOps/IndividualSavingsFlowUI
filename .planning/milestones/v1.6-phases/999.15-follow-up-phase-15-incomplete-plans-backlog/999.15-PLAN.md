# Phase 999.15: Follow-up — Phase 15 incomplete plans (BACKLOG) - Plan

## Goal Description

이 페이즈는 Phase 15 실행 과정에서 미처 작성되지 못한 `15-SUMMARY.md`를 작성하여 Phase 15를 최종적으로 마감하며, 사용자가 제보한 핵심 결함인 계좌 연결(accountId) 유실 문제를 해결하고, `npm run build` 시 패치 버전을 자동으로 범프(Patch Version Bump)하는 자동화 시스템을 구축하는 것을 목표로 합니다.

구체적인 요구사항은 다음과 같습니다:
1. **계좌(accountId) 변경 데이터 정상 저장(D-01, D-02):**
   - `apps/step1/app.js` 내의 `handleItemInput` 함수에서 `field === "accountId"` 분기 처리가 누락되어 발생한 오동작을 수정하여, 사용자가 드롭다운에서 계좌를 선택하고 완료했을 때 `item.accountId`가 정상적으로 데이터 모델에 반영되도록 조치합니다.
   - 드롭다운 `<select>` 태그의 변경 감지가 보다 명확하게 연동될 수 있도록, `input` 이벤트뿐만 아니라 `change` 이벤트도 연계 바인딩할 수 있는지 검토하여 연동 안정성을 극대화합니다.
2. **빌드 시 자동 패치 버전 범프(D-03, D-04):**
   - `npm run build` 실행 시 자동으로 `package.json`의 패치 버전(Patch Version, x.y.z의 z)을 1만큼 증가시키는 `scripts/bump-version.js` 스크립트를 작성합니다.
   - 버전 증가가 완료된 후 기존 `sync-version.js`가 뒤이어 동작하여 `manifest`, `sw.js`, `utils.js` 등 동기화 파일에 올라간 버전을 즉각 주입하도록 `"build"` 스크립트를 변경합니다.
3. **Phase 15 최종 마무리 및 SUMMARY 작성(D-05):**
   - `.planning/phases/15-chart-enhancement-responsive/15-SUMMARY.md` 파일을 생성하여 Phase 15의 본래 계획과 이번 백로그 개선 결과를 종합 정리 및 마감합니다.
