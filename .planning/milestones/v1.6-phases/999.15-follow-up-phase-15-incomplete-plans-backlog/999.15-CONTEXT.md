<decisions>
## Implementation Decisions

### 계좌(accountId) 연결 기능 정상화
- **D-01:** `apps/step1/app.js` 내의 `handleItemInput` 함수에서 `field === "accountId"` 분기 처리가 누락되어 발생한 오동작을 수정하여, 사용자가 드롭다운에서 계좌를 선택하고 완료했을 때 `item.accountId`가 정상적으로 데이터 모델에 반영되도록 조치합니다.
- **D-02:** `<select>` 태그의 변경 감지가 보다 명확하게 연동될 수 있도록, `input` 이벤트뿐만 아니라 `change` 이벤트도 연계 바인딩할 수 있는지 검토하여 연동 안정성을 극대화합니다.

### 빌드 시 자동 패치 버전 범프
- **D-03:** `npm run build` 실행 시 자동으로 `package.json`의 패치 버전(Patch Version, x.y.z의 z)을 1만큼 증가시키는 `scripts/bump-version.js` 스크립트를 작성합니다.
- **D-04:** 버전 증가가 완료된 후 기존 `sync-version.js`가 뒤이어 동작하여 `manifest`, `sw.js`, `utils.js` 등 동기화 파일에 올라간 버전을 즉각 주입하도록 `"build"` 스크립트를 변경합니다.

### Phase 15 최종 마감
- **D-05:** 원래 Phase 15의 계획(`15-PLAN.md`)과 구현 상황을 종합적으로 대조 평가하여 `15-SUMMARY.md` 파일을 작성하고 아카이빙합니다.

### the agent's Discretion
- `scripts/bump-version.js` 의 파일 입출력 및 파싱 로직의 디테일과 `package.json` 의 포맷 정밀도 보존 작업은 개발 에이전트의 재량에 위임합니다.

</decisions>
