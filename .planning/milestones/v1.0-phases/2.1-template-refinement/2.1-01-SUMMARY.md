# Phase 2.1 Summary: 템플릿 데이터 고도화 및 실용성 강화

## 개요
프리셋 템플릿의 데이터 품질을 고도화하여 단일 항목(생활비 등) 대신 현실적인 12대 가계부 항목(주거비, 식비, 비상금, 연금 등)을 기반으로 한 고해상도 자산 흐름을 생성하도록 구현했습니다.

## 구현 상세
- `presets.js` 내에 카테고리별 세부 배분 비율 정의 (`EXPENSE_DETAIL`, `SAVINGS_DETAIL`, `INVEST_DETAIL`).
- 각 카테고리별 상수 테이블에는 가중치(weight)를 포함하여 연봉(income) 수준에 따른 자동 금액 산출 로직 적용.
- `applyPreset` 함수를 개편하여 `distributeAmount` 헬퍼를 통해 총액을 세부 항목으로 분배하고, 반올림 오차를 첫 번째 항목에서 보정하도록 처리.
- 기존 `constants.js`의 `DEFAULT_EXPENSE_ITEMS` 등의 ID 체계를 준용하여 렌더러/에디터와의 호환성 확보.

## 검증 결과
- [x] 지출 세부 항목 6개(주거비, 관리비, 통신비, 교통비, 식비, 기타생활비) 생성 완료.
- [x] 저축 세부 항목 3개(비상금, 청년적금, 주택청약) 생성 완료.
- [x] 투자 세부 항목 3개(해외주식, ISA, 개인연금/IRP) 생성 완료.
- [x] 항목별 합산액이 초기 할당 총액과 정확히 일치함 (오차 보정 로직 작동).
- [x] 모든 항목에 고유한 ID가 부여됨.

## 배움 및 이슈
- 반올림 오차(`Math.round`)로 인해 산출된 세부 항목의 합계가 카테고리의 원본 총액과 달라지는 현상을 발견했으며, 오차만큼을 배열의 첫 번째 항목(예: 주거비, 비상금)에 합산/차감하는 방식으로 무결성을 확보했습니다.
- 외부 API(app.js 등)를 수정하지 않고 `applyPreset` 반환값의 배열 내용만 고해상도로 교체하여 기존 Sankey 및 Editor 컴포넌트가 새로운 템플릿을 완벽히 소화할 수 있었습니다.
