# Phase 2.1 Plan: 프리셋 데이터 고도화

## 목표
`presets.js`의 `applyPreset` 함수가 반환하는 데이터를 고도화하여, 프리셋 적용 시 단일 항목("생활비", "저축", "투자") 대신 현실적인 가계부 항목 세트(주거비, 식비, 통신비, 비상금, 적금, 주식, 연금 등)를 생성하도록 한다.

## 제약 조건
- 수정 대상은 `apps/step1/modules/presets.js` **단일 파일**에 한정한다.
- `applyPreset` 함수의 반환 인터페이스(키 구조)는 변경하지 않는다 — 하위 호환 유지.
- `PRESET_SALARIES` 배열은 변경하지 않는다 (이미 적절함).
- `PRESET_STYLES`의 상위 비율(expenseRate, savingsRate, investRate)은 유지하되, 내부 세부 배분 로직을 추가한다.
- 모든 금액은 **원(Won)** 단위를 유지한다.
- 세부 항목 ID는 기존 `constants.js`의 `DEFAULT_*_ITEMS` ID와 일관성을 유지한다.

<read_first>

</read_first>

<action>
`presets.js`에 다음 3개의 상수 배열을 추가한다. 각 항목은 `{ id, name, weight }` 형태이며, `weight`는 해당 카테고리 총액 대비 비율이다. weight의 합은 정확히 1.0이어야 한다.

```javascript
const EXPENSE_DETAIL = [
  { id: "rent",        name: "주거비(월세)", weight: 0.35 },
  { id: "maintenance", name: "관리비",       weight: 0.06 },
  { id: "telecom",     name: "통신비",       weight: 0.04 },
  { id: "transport",   name: "교통비",       weight: 0.08 },
  { id: "food",        name: "식비",         weight: 0.32 },
  { id: "etc",         name: "기타생활비",   weight: 0.15 },
];

const SAVINGS_DETAIL = [
  { id: "emergency",            name: "비상금",     weight: 0.30, annualRate: 2.0 },
  { id: "youth-saving",         name: "청년적금",   weight: 0.50, annualRate: 3.6 },
  { id: "housing-subscription", name: "주택청약",   weight: 0.20, annualRate: 2.9 },
];

const INVEST_DETAIL = [
  { id: "global-stock", name: "해외주식",     weight: 0.40 },
  { id: "isa",          name: "ISA",          weight: 0.35 },
  { id: "pension",      name: "개인연금/IRP", weight: 0.25 },
];
```

- 지출 ID는 `constants.js`의 `DEFAULT_EXPENSE_ITEMS`와 동일하게 맞춘다 (rent, maintenance, telecom, transport, food, etc).
- 저축 ID는 `DEFAULT_SAVINGS_ITEMS`와 일관성을 유지하되 비상금(emergency)을 추가한다.
- 투자 ID는 `DEFAULT_INVEST_ITEMS`에서 gold-spot 대신 pension(개인연금/IRP)으로 교체한다.
</action>

## Task 2: `applyPreset` 함수 개편 — 세분화된 항목 배열 생성

<read_first>
- `apps/step1/modules/presets.js` (Task 1에서 추가한 상수 포함)
- `apps/step1/modules/constants.js` (DEFAULT_INPUTS 구조 참조 — 반환값이 이 구조와 merge되므로 키가 일치해야 함)
- `apps/step1/modules/calculator.js` (buildMonthlySnapshot에서 expenseItems/savingsItems/investItems를 어떻게 소비하는지 확인)
</read_first>

<action>
`applyPreset` 함수의 반환 로직을 다음과 같이 수정한다:

1. 기존과 동일하게 `income`, `expense`, `savings`, `invest` 총액을 먼저 산출한다.
2. 총액을 세부 항목별 weight로 배분한다. 각 항목의 `amount`는 `Math.round(총액 * weight)`로 계산한다.
3. 반올림 오차를 보정한다: 세부 항목 amount 합계와 카테고리 총액의 차이를 첫 번째 항목(지출의 경우 "주거비", 저축의 경우 "비상금", 투자의 경우 "해외주식")에 가감한다.
4. 반환 객체의 `expenseItems`, `savingsItems`, `investItems`에 세분화된 배열을 담는다.
5. 저축 항목에는 `annualRate` 필드를 포함한다.

구체적으로 다음과 같은 헬퍼 함수를 추가한다:

```javascript
function distributeAmount(total, details) {
  const items = details.map(d => ({
    ...d,
    amount: Math.round(total * d.weight),
  }));
  // 반올림 오차 보정: 첫 번째 항목에 가감
  const sum = items.reduce((s, i) => s + i.amount, 0);
  items[0].amount += (total - sum);
  // weight 필드 제거 (state에 불필요)
  return items.map(({ weight, ...rest }) => rest);
}
```

`applyPreset` 함수의 return 문을 다음으로 교체한다:

```javascript
return {
  incomes: [{ id: "income-preset", name: "급여", amount: income }],
  monthlyExpense: expense,
  monthlySavings: savings,
  monthlyInvest: invest,
  expenseItems: distributeAmount(expense, EXPENSE_DETAIL),
  savingsItems: distributeAmount(savings, SAVINGS_DETAIL),
  investItems: distributeAmount(invest, INVEST_DETAIL),
  monthlyDebtPayment: 0,
  startCash: 0,
  startSavings: 0,
  startInvest: 0,
  startDebt: 0,
};
```
</action>

<acceptance_criteria>
- `applyPreset(50000000, 'neutral')` 호출 시 `expenseItems` 배열의 길이가 6이다.
- `applyPreset(50000000, 'neutral')` 호출 시 `savingsItems` 배열의 길이가 3이고, 각 항목에 `annualRate` 필드가 존재한다.
- `applyPreset(50000000, 'neutral')` 호출 시 `investItems` 배열의 길이가 3이다.
- `expenseItems`의 각 항목에 `id`, `name`, `amount` 키가 존재한다.
- `expenseItems.reduce((s,i) => s+i.amount, 0)` === `monthlyExpense` (반올림 오차 보정 확인).
- `savingsItems.reduce((s,i) => s+i.amount, 0)` === `monthlySavings`.
- `investItems.reduce((s,i) => s+i.amount, 0)` === `monthlyInvest`.
- 모든 항목의 `id` 값이 고유하다 (중복 없음).
- `distributeAmount` 함수가 `presets.js` 내에 존재한다.
</acceptance_criteria>
