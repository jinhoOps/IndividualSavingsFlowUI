## Accomplishments

- Added Playwright coverage for the Phase 10.5 integrated modal shell, including exact tab order, rail labels, no nested modal navigation, neutral-copy guardrails, and no horizontal overflow at 1280px, 768px, and 390px.
- Reworked `financial-modal-controller.js` so the existing `재무설정 상세` modal holds one integrated draft state across income, living expense, investment, and savings categories.
- Added an always-visible rail with `월 수입`, `월 생활비`, `월 투자`, `자동 저축`, and `상태`, rendered through DOM APIs and `textContent`.
- Added flat responsive CSS for the tablist, rail, and result panel while preserving the existing save/cancel and create-item modal behavior.

## Decisions Made

- Kept the implementation inside the existing controller instead of adding a helper module, because Plan 02 only changes the modal shell and rail while preserving the existing persistence boundary.
- Used `data-financial-detail-*` selectors for tests and future plans so the shell contract is stable without coupling tests to private controller variables.
- Kept automatic-savings adjustment as a visible reserved action only when future commitments exceed monthly income; Plan 10.5-04 owns the detailed adjustment choices and save gate.
