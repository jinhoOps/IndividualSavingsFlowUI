## 🔬 테스트 시나리오 목록

### [UAT-01] 계좌(통장) 프리셋 노출 및 CRUD 기능 검증
* **목적:** 사용자가 통장 목록을 직접 제어하고, 초기 프리셋이 올바르게 구성되는지 검증합니다.
* **수행 단계:**
  1. 앱 최초 진입 또는 설정 초기화(설정 -> 데이터 초기화)를 수행합니다.
  2. 좌측 또는 상단 통장 관리 영역에 `급여계좌`, `생활비계좌`, `주식계좌` 3개의 기본 프리셋이 정상 노출되는지 확인합니다.
  3. '항목 편집'을 눌러 새로운 계좌를 추가하고 이름을 '비상금통장'으로 변경한 후 적용합니다.
* **기대 결과:**
  - 초기 진입 시 기본 계좌 3개가 누락 없이 표시되어야 합니다.
  - 신규 계좌 추가 및 이름 편집이 오류 없이 적용되고 목록에 표시되어야 합니다.
* **테스트 결과:** 🟡 대기 중 (Pending)

---

### [UAT-02] 항목 편집 시 계좌 연동 및 저장 무결성 검증 (Phase 999.15 핫픽스 반영)
* **목적:** 수입/지출/저축/투자 상세 편집 시 계좌가 정상 바인딩되고, 변경사항 저장 시 `accountId` 데이터가 정상 영속화되는지 검증합니다.
* **수행 단계:**
  1. 수입/지출/저축/투자 중 하나의 항목 옆 '편집' 버튼을 클릭합니다.
  2. 계좌 드롭다운 옵션에 새로 추가한 '비상금통장'을 포함해 모든 계좌가 정확히 노출되는지 확인합니다.
  3. 아이템의 연결 계좌 드롭다운을 변경한 후 상단의 '변경 적용'을 눌러 저장합니다.
  4. 화면을 새로고침하거나 세부를 재진입하여, 변경한 계좌 정보가 유지되는지 확인합니다.
* **기대 결과:**
  - 계좌 드롭다운에 모든 계좌 리스트가 정상 표기되어야 합니다.
  - `handleItemInput` 처리 시 `accountId`가 누락 없이 올바르게 데이터 구조에 반영되고 영속화되어야 합니다.
* **테스트 결과:** 🟡 대기 중 (Pending)

### [UAT-03] 잉여현금 자동 이체 배너 및 계좌 변경 검증
* **목적:** 수입이 지출보다 클 때 잉여현금 이체 배너가 적절히 활성화되고, 대상 계좌 변경 시 차트에 실시간 연동되는지 검증합니다.
* **수행 단계:**
  1. 수입(예: 300만원)을 입력하고 지출/저축/투자 합계를 수입보다 적게 설정하여 잉여현금(예: 50만원)을 발생시킵니다.
  2. 요약 카드 하단에 잉여현금 안내 배너(`💡 이번 달 남는 잉여현금...`)가 렌더링되는지 확인합니다.
  3. 배너 안의 드롭다운을 통해 이체 대상 계좌를 '생활비계좌' 등으로 변경하고 '변경 적용'을 누릅니다.
  4. Sankey 차트에서 잉여현금(`surplus`) 링크의 유입이 변경한 계좌로 연결되는지 관찰합니다.
* **기대 결과:**
  - 잉여현금 존재 시에만 이체 안내 배너가 정확히 표시되어야 합니다.
  - 이체 대상 계좌를 변경 시 즉시 차트 노드 연결선(Sankey Link)이 수정되어 렌더링되어야 합니다.
* **테스트 결과:** 🟡 대기 중 (Pending)

### [UAT-04] Sankey 차트 4단계 레이어 흐름 및 무결성(Nyquist) 검증
* **목적:** 계좌 노드가 중간 레이어로 삽입된 Sankey 차트의 흐름 정합성과 반응형 레이아웃을 검증합니다.
* **수행 단계:**
  1. Sankey 차트가 `수입원 -> 계좌 노드 -> 세부 항목` 순으로 레이어가 구성되어 흐르는지 시각적으로 검증합니다.
  2. 계좌 간 내부 자금 이동(예: 급여계좌 -> 생활비계좌) 링크가 누락 없이 출력되는지 확인합니다.
  3. 모든 노드에 들어오는 값의 합과 나가는 값의 합이 일치하여 흐름이 도중에 끊어지거나 어긋나지 않는지 확인합니다.
  4. 브라우저 창 크기를 760px 이하로 줄여 모바일 레이아웃에서 차트 비율이 깨지지 않는지 확인합니다.
* **기대 결과:**
  - 4단계 레이어로 차트가 정상 렌더링되며, 유입/유출량이 정확히 일치(Flow 무결성)해야 합니다.
  - 모바일 해상도에서도 노드나 텍스트가 겹쳐서 가독성이 저해되지 않아야 합니다.
* **테스트 결과:** 🟡 대기 중 (Pending)

### [UAT-05] 빌드 자동 버전 범프 및 이전 데이터 하이드레이션 검증
* **목적:** 빌드 자동화 스크립트 동작 및 하위 호환성을 검증합니다.
* **수행 단계:**
  1. 프로젝트 루트에서 `npm run build` 명령을 실행합니다.
  2. 빌드가 정상 완료된 후 `package.json` 파일의 `version` 정보 패치 자리가 1 증가했는지 확인합니다.
  3. 구버전 데이터(계좌 속성이 누락된 예전 로컬스토리지 백업 데이터)를 데이터 허브를 통해 복원해보거나 초기화 로드를 강제해 봅니다.
* **기대 결과:**
  - 빌드 과정이 에러 없이 끝나야 하며 버전이 자동 범프되어야 합니다.
  - 이전 데이터를 불러왔을 때 에러가 발생하지 않고, 기본 계좌가 하이드레이션되며 정상 복구되어야 합니다.
* **테스트 결과:** 🟢 Pass (에러 없는 빌드 및 자동 패치 버전 0.11.2 ➔ 0.11.3 범프 및 싱크 파일 업데이트 확인 완료)
