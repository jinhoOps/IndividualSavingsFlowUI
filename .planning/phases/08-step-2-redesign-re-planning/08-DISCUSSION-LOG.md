| Option | Description | Selected |

## 입력 단순화 방식

| Option | Description | Selected |
|--------|-------------|----------|
| 초기 투자금 + 월 투자금 + 기간 | 사용자가 실제로 아는 돈과 기간만 먼저 입력 | ✓ |
| 월 투자금 + 기간만 | 초기 투자금은 0 또는 Step 1 연동값으로 처리 | |
| 전략 프리셋 + 월 투자금 + 기간 | SCHD/JEPI/QQQI 같은 전략 선택을 앞에 배치 | |

**User's choice:** 초기 투자금 + 월 투자금 + 기간.
**Notes:** 전략 가정은 프리셋 카드 + 기본 닫힘 고급 설정. 전략 카드 분류는 성장형/배당성장/커버드콜. Step 1 원본 월 투자 가능 금액은 자동으로 가져오되 Step 2에서만 수정 가능해야 하며, Step 2 초기화 시 Step 1 원본값을 다시 가져와야 한다. 현재 자동 적용 + 수정 가능 기능이 동작하지 않는 문제도 Phase 08 범위.

| Option | Description | Selected |
|--------|-------------|----------|

| Option | Description | Selected |
|--------|-------------|----------|
| 초기 투자금만 | 현재 투자 가능한 목돈이 5천만 원 이하이면 경고 | ✓ |
| 초기 투자금 + 1년치 월 투자금 | 곧 운용할 실제 투자 규모 기준 | |

## ETF/전략 수치의 현실성

| Option | Description | Selected |
|--------|-------------|----------|
| 보수적 고정값 | 앱 안에 보수적인 기본값을 넣고 예시 가정임을 명시 | ✓ |
| 사용자 직접 조정 중심 | 사용자가 배당률/성장률/상승률을 직접 입력 | |
| 최근 시장 데이터 기반 | 최신 수치를 반영하지만 정적 앱 유지보수 부담 증가 | |

**User's choice:** 보수적 고정값. 단, 전략 수치는 프로젝트에 있는 백데이터 CSV 정리도 필요함.
**Notes:** `public/data/indices/*.json`와 정리된 백데이터를 기반으로 보수값/보수 범위를 만든다. 화면 표시는 범위 표시를 사용하고, 실제 계산은 보수 기본값으로 하되 고급 설정에서 수정 가능하게 한다. 루트의 `qqq_raw.csv`, `qqq_daily_raw.csv`, `qqq_daily_stooq.csv` 정리를 Phase 08에 포함한다.

## 모바일 화면 구조

| Option | Description | Selected |
|--------|-------------|----------|
| 판단 문장 → 입력 → KPI → 그래프 → 비교 카드 → 상세 보기 | 먼저 비교 이유를 말하고 입력 뒤 결과를 표시 | ✓ |
| 입력 → KPI → 판단 문장 → 그래프 → 비교 카드 → 상세 보기 | 계산부터 시작 | |
| KPI → 그래프 → 입력 접힘 → 비교 카드 → 상세 보기 | 결과 중심이지만 입력이 뒤로 밀림 | |

**User's choice:** 판단 문장 → 입력 → KPI → 그래프 → 비교 카드 → 상세 보기.
**Notes:** 모바일 비교 카드는 세로 고밀도 카드 3개. 기간 세그먼트 + 터치 툴팁을 유지하고, 피드백은 랜덤 애니메이션 느낌을 주되 결과 오해나 레이아웃 흔들림이 없는 마이크로 인터랙션으로 제한한다. 상세 테이블은 접힌 상세 + 가로 스크롤 유지에 요약카드를 추가한다.
