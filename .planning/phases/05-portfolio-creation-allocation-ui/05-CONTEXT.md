# Phase 5 Context: Portfolio Creation & Target Allocation UI

## Domain Boundary
이 페이즈는 `apps/step3` 공간을 활용하여 **"나만의 적립식 포트폴리오 만들기"** 서비스의 핵심 입력 및 비중 편집 UI/UX를 개발합니다. 사용자는 포트폴리오의 이름, 주기, 총 매수 금액, 종목 구성 및 금액을 유연하게 설정할 수 있습니다.

## Locked Requirements
- **PORT-01**: 나만의 포트폴리오 만들기 화면 구현 (추가하기 버튼, 종목명 입력, 최소 2개 이상 종목 선택 강제, 포트폴리오 이름 지정)
- **PORT-02**: 적립식 주기(매일/매주/매달) 및 매수 금액 설정 (주식당 최소 1,000원 제한, 상단에 총금액 및 종목 개수 표기, 각 종목별 % 비중 실시간 표기 및 비중 편집 팝업/토글 제공)

## Implementation Decisions

### 1. 비중 편집 및 금액 합산 로직
- **금액 기반 자동 합산**: 사용자는 개별 종목의 매수 금액(원)만 적습니다. 총 매수 금액을 따로 입력할 필요가 없으며, 시스템이 개별 종목 금액의 합을 자동으로 실시간 계산하여 **'총 매수 금액'**으로 자동 표기해 줍니다.
- **실시간 비중(%) 도출**: 개별 종목 금액 입력에 의해 총 매수 금액이 결정되면, 시스템이 즉시 `(개별 종목 금액 / 총 매수 금액) * 100` 연산을 수행하여 각 종목의 비중 %를 화면에 실시간으로 정수(반올림) 표기해 줍니다.
- **유효성 검증 규칙**:
  - 개별 종목의 금액은 **최소 1,000원 이상**이어야 하며, 반드시 **1,000원 단위**(예: 1,000원, 2,000원, 10,000원 등 / 1,500원이나 2,300원 등은 입력 불가)로 입력해야 합니다.
  - 종목이 최소 2개 이상이고, 모든 종목의 금액이 이 제약조건(최소 1,000원 이상 및 1,000원 단위)을 완벽히 충족할 때만 '확인' 버튼이 활성화됩니다.
  - 상단 혹은 합산 영역에 `"총 N개의 주식 | 총 매수 금액: X원"` 형태로 실시간 합산 정보가 표기됩니다.

### 2. 주기 및 총 매수 금액 입력 UI
- **가로형 프리미엄 세그먼트 컨트롤**: 주기 선택(매일, 매주, 매달)은 에디토리얼 테마의 가로형 세그먼트 버튼 컴포넌트를 사용합니다.
- **실시간 한글 금액 힌트**: 각 개별 종목의 금액 입력창 하단, 그리고 자동 합산된 '총 매수 금액' 영역 옆에 `IsfUtils.convertToKoreanWon` 실시간 한글 금액 힌트가 노출됩니다.

### 3. 포트폴리오 목록 카드 레이아웃
- **에디토리얼 포트폴리오 요약 카드**: 생성된 포트폴리오는 메인 상단 영역에 미려한 카드 형태로 리스팅됩니다.
- **카드 표기 정보**: 포트폴리오 이름, 설정 주기, 총 매수 금액, 구성된 종목 개수 요약.
- **인터랙션**: 마우스 호버 시 Sunset Orange 테두리 및 Y축 -2px 부유 효과가 적용되며, 카드 클릭 시 구성 종목의 상세 금액/비중 및 누적 투자액 추이를 보여주는 세부 내역 팝업/모달이 연동됩니다.

## Code Context
- **기존 모듈 활용**: `apps/step3/modules/dom.js`, `apps/step3/modules/state.js`, `apps/step3/modules/calculator.js` 구조를 보호하되, 포트폴리오 리밸런싱 목적에서 **적립식 포트폴리오 관리** 목적으로 상태 관리자 및 계산 엔진을 리팩토링합니다.
- **글로벌 유틸리티**: `shared/core/utils.js`의 `IsfUtils.convertToKoreanWon`, `IsfUtils.updateAllKoreanWonHints`를 활용하여 힌트 상시 표기를 유지합니다.
