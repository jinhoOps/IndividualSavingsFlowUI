# Phase 17: Step 1 Income Flow & Wallet Partitioning - Validation

## Verification Checklist

본 문서는 Phase 17 구현 완료 후 기능 동작의 무결성과 시각적/로직적 신뢰성을 측정하고 최종 보증하기 위한 검증 시나리오 명세서입니다.

---

### 1. 계좌(통장) CRUD 및 입력 UI 검증
- **검증 목적:** 사용자가 통장 목록을 직접 제어하고, 각 수입/지출/저축/투자 세부 항목의 편집 모드에서 계좌 매핑이 올바르게 바인딩되는지 검증합니다.
- **검증 시나리오:**
  1. **초기 진입 프리셋 확인:**
     - 앱 최초 진입 또는 설정 초기화 시, 통장 관리 섹션에 `급여계좌`, `생활비계좌`, `주식계좌` 3개의 프리셋이 정상 노출되는지 확인합니다.
  2. **계좌 추가 및 이름 편집:**
     - '항목 편집'을 눌러 새로운 계좌를 추가하고 이름을 '비상금통장'으로 변경한 후 적용합니다.
     - 통장 목록에 '비상금통장'이 추가되었고, 편집 모드 외에서도 텍스트가 깨지지 않는지 확인합니다.
  3. **세부 항목별 드롭다운 연동 및 매직 매핑:**
     - 수입/지출/저축/투자 상세 편집기를 실행합니다.
     - 각 아이템 행에 새롭게 추가한 '비상금통장'을 포함한 모든 계좌 리스트가 `<select>` 옵션에 정확히 나타나는지 확인합니다.
     - 새로 추가하는 아이템에 대해 매직 매핑(지출 ➔ 생활비계좌, 투자 ➔ 주식계좌, 수입 ➔ 급여계좌)이 자동으로 매핑되는지 검증합니다.
     - 드롭다운을 변경하여 적용했을 때, 아이템의 매핑 계좌 정보가 정상 저장 및 유지되는지 확인합니다.

---

### 2. 잉여현금 자동 이체 문장형 UI 검증
- **검증 목적:** 잉여현금 발생 시 요약 카드 하단 배너가 정상 노출되고, 드롭다운 변경이 시뮬레이션 및 차트에 양방향 반영되는지 검증합니다.
- **검증 시나리오:**
  1. **배너 노출 여부:**
     - 수입(300만원) > 지출/저축/투자 합계(250만원) 상태를 만들어 잉여현금 50만원을 발생시킵니다.
     - 요약 카드 하단에 `"💡 이번 달 남는 잉여현금 [50만 원]은 [주식계좌 ▾]로 자동 이체합니다."` 배너가 정상 활성화되는지 확인합니다.
     - 지출을 늘려 잉여현금이 0이 되거나 결손이 발생했을 때, 해당 배너가 즉시 화면에서 사라지는지 검증합니다.
  2. **이체 대상 계좌 변경:**
     - 배너 내 드롭다운을 열어 이체 대상 계좌를 `주식계좌`에서 `생활비계좌`로 변경합니다.
     - 변경 즉시 상단에 '저장 대기 중인 변경사항' 펜딩 바가 나타나는지 확인하고, '변경 적용'을 클릭해 저장합니다.
     - Sankey 차트 상에서 잉여현금(`surplus` 노드)으로 유입되는 선의 시작점이 기존 `주식계좌` 노드에서 `생활비계좌` 노드로 실시간 재렌더링되는지 확인합니다.

---

### 3. 중간 계좌 레이어 Sankey 차트 흐름 & 무결성 검증
- **검증 목적:** 대분류 대신 계좌 노드가 중간 레이어로 적용된 Sankey 차트가 완벽한 유입/유출 정합성을 유지하며 렌더링되는지 검증합니다.
- **검증 시나리오:**
  1. **4단계 레이어 구조 확인:**
     - Sankey 차트 상에서 흐름이 `[수입원(급여 등)] ➔ [계좌 노드들(급여/생활비/주식계좌)] ➔ [세부 항목들(식비/해외주식 등)]` 구조로 깔끔하게 렌더링되는지 검증합니다.
  2. **계좌 간 이체(내부 링크) 생성 및 가독성:**
     - 수입은 `급여계좌`로 입금되었으나, 실제 지출은 `생활비계좌`에서 발생하는 구조일 때, `급여계좌 ➔ 생활비계좌` 로 자금이 이동하는 연결 링크가 차트에 자동 생성 및 렌더링되는지 점검합니다.
     - 마우스를 올렸을 때 툴팁에 이체 금액이 정확히 표시되는지 확인합니다.
  3. **Sankey 무결성 검사 (Nyquist):**
     - 각 계좌 노드에 진입하는 모든 흐름의 합(Inflow)과 나가는 모든 흐름의 합(Outflow)이 정확하게 일치하여 차트의 일부가 공중에 붕 뜨거나 끊어지는 현상이 없는지 시각적으로 확인합니다.
     - 760px 이하 모바일 디바이스 뷰포트에서 노드 겹침 현상 없이 차트 비율이 깔끔하게 유지되는지 개발자 도구로 재확인합니다.

### 4. 빌드 시스템 및 스토리지 하위 호환성 검증
- **검증 목적:** 빌드 스크립트 실행 시 스타일 및 로직 빌드가 무결하게 진행되며, 기존 스토리지 데이터에 신규 스펙이 유연하게 접합되는지 확인합니다.
- **검증 시나리오:**
  1. **Vite 프로덕션 빌드:**
     - 터미널에서 `npm run build`를 수행하여 트랜스파일링이나 번들링 과정에서 에러 없이 성공적으로 `dist/` 폴더가 빌드되는지 검증합니다.
  2. **구버전 데이터 마이그레이션:**
     - 브라우저 로컬 스토리지에 `accounts` 속성이 결여된 기존 데이터가 로드될 때, `input-sanitizer.js`가 에러를 뿜지 않고 기본 프리셋 계좌를 안전하게 하이드레이션하며 기본 매직 매핑을 강제 적용해 하위 호환성을 완벽히 충족하는지 테스트합니다.
