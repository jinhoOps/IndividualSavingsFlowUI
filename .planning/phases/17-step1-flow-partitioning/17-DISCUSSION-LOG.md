## Sankey 차트 상의 중간 계좌 레이어 시각화 방식

| Option | Description | Selected |
|--------|-------------|----------|
| 3-A. 5단계 레이어 Sankey 확장 | `수입원 -> 총수입 -> 중간 계좌 -> 대분류 -> 세부 항목`으로 확장 (UX 토글 보완 시도했으나 복잡성 제기되어 철회) | |
| 3-B. 계좌 중심 4단계 Sankey | 대분류 노드를 아예 생략하고 `수입원 -> 중간 계좌 -> 세부 항목`으로 흐름 단순화 | |
| **대안 A. 계좌와 대분류의 융합 (최종 채택)** | 기존 고정 대분류 노드 자체를 사용자가 정의한 '계좌/통장' 노드로 대체하여 4단계 레이어 구조 유지 | ✓ |

**User's choice:** 대안 A (계좌와 대분류의 융합)
**Notes:** 5단계 레이어로의 확장은 차트를 지나치게 복잡하게 만들고, 온오프 토글을 추가하는 것조차 또 다른 인지 부하(UX 복잡도)가 되므로, 기존 대분류 노드 자체를 계좌 노드로 치환하여 4단계 차트 레이어를 깔끔하게 보존하기로 합의함.
