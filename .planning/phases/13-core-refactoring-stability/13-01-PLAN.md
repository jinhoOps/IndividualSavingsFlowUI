# 단위 정합성 점검 및 과세 경고 로직 검증

<objective>
STAB-01(단위 정합성)과 STAB-02(과세 경고 로직)를 점검하여 현재 코드의 정합성을 확인하고, 발견된 문제만 수정합니다. RESEARCH.md에 따르면 현재 상태는 양호하므로 확인 위주의 작업입니다.

## Goal-Backward Verification
- truth: 모든 UI 표시 금액은 만원 단위이며, 모든 저장/계산 데이터는 원 단위를 유지한다
- truth: 직접 산술 연산(* 10000, / 10000)은 마이그레이션 함수 내부에만 존재한다
- truth: IsfUtils.toWon, IsfUtils.toMan 헬퍼가 단위 변환의 유일한 경로이다
- truth: getFinancialIncomeStatus의 임계값(1,900만 원 / 3,400만 원)이 명명된 상수로 정의되어 있다
- truth: 과세 경고가 연간 금융소득 1,900만 원 초과 시 warn, 3,400만 원 초과 시 crit를 반환한다
</must_haves>
