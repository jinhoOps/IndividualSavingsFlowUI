# Phase 10.6: Financial Detail Modal Editing UX Repair - Context

**Gathered:** 2026-06-24
**Status:** Seeded for later discussion

## Goal

Repair the existing `재무설정 상세` modal UX without reopening the Phase 10.5 decision to remove couple/self/spouse UI. Phase 10.6 should make the modal feel compact in read mode, predictable during row-level edits, and clear about when draft changes are pending.

## User Decisions

- Use a new Phase 10.6 rather than reopening Phase 10.5.
- Keep Phase 10.5's removal of `부부합산`, `본인 설정`, `배우자 설정`, spouse income editing, and couple/household-status UI.
- Do not restore a separate `#householdBudgetPanel`.
- Default modal state is compact read mode.
- Editing is row-level: one row expands for editing while other rows remain in read mode.
- Clicking outside the edited row folds it back to read mode.
- If values changed, the draft remains and the pending bar appears; nothing is persisted until the pending bar action.
- Existing values must remain in the input when editing starts. Editing must feel like modifying an existing row, not recreating it.
- Money controls:
  - direct input remains available
  - `+` / `-` stepper changes by 10,000 KRW
  - quick buttons are increase-only: `+5만`, `+10만`, `+100만`
  - values must not go below 0
- Pending bar copy is `취소` / `적용`.
- Pending bar appears only after actual draft changes.
- Pending bar `취소` reverts draft changes and keeps the modal open.
- Pending bar `적용` persists draft changes and keeps the modal open.
- Modal close is separate from pending-bar actions.
- `새 항목 추가` belongs on the right side of each tab's item header, not as a detached/floating button. On mobile it may wrap below the header.

## Problems To Fix

- `<!-- 신설 모달 전용 플로팅 펜딩 바 -->` / "신설 모달" implementation language should not leak into the user-facing UX or design model.
- `financial-detail-panel__title` duplicates tab names. Tab labels are enough; repeated panel titles add noise.
- Read-mode rows are not compressed enough and can wrap because of icons or repeated labels.
- Labels such as `수입, 수입` and `변동비, 변동비` feel duplicated.
- The current fixed `새 항목 추가` location is detached from the panel where new rows appear.
- The pending bar appears too eagerly when entering edit mode rather than only after a value changes.
- `취소` / edit actions must not close the whole `재무설정 상세` modal; they should only resolve edit/pending state.

## Scope

In scope:
- `재무설정 상세` modal read-mode density and row layout.
- Row-level edit entry/exit behavior.
- Existing-value money editing with steppers and quick-add buttons.
- Pending-change bar state and copy.
- Header-local add-item controls.
- Tests for draft/pending behavior and no modal-close regressions.

Out of scope:
- Couple/self/spouse UI.
- Separate household budget panel restoration.
- Phase 11 spending capture.
- Phase 12 dual-flow merge.
- Phase 14 real-estate affordability.
- Replacing the sanitizer, persistence, or Sankey architecture.
