## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| category card/modal input -> draft state | User-controlled names, groups, aliases, and amounts enter modal state. |
| modal confirmation -> sanitizer/persistence | Draft rows become durable Step 1 inputs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07 | Tampering | `financial-modal-controller.js` | mitigate | Reuse existing amount/name validation and commit only through sanitizer/persistence. |
| T-09-08 | Information Disclosure | summary cards | accept | Cards display local user financial data already present in Step 1. |
| T-09-09 | Tampering / XSS | summary/modal rendering | mitigate | Render user-controlled names with `textContent` or `IsfUtils.escapeHtml`; build select options with DOM APIs. |
| T-09-SC | Tampering | npm installs | mitigate | No package-manager installs are part of this plan; keep external dependency set unchanged. |
</threat_model>
