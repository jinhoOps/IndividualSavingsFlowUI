<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Modal item edit -> persisted financial model | User-entered names, amounts, groups, and account ids are committed to local persisted inputs. |
| Legacy transfer data -> current Sankey output | Older saved states may still contain transfer rules even when the UI no longer exposes editing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-13 | Tampering | financial modal compact editor | mitigate | Keep all commits through existing sanitizer/persistence path and preserve explicit save/cancel. |
| T-09-14 | XSS | group custom input | mitigate | Assign all labels/options through DOM APIs and `textContent`/form values only. |
| T-09-15 | Denial of Service | legacy transfer data | mitigate | Ignore or tolerate old `transfers` arrays without throwing when the editor DOM is absent. |
</threat_model>
