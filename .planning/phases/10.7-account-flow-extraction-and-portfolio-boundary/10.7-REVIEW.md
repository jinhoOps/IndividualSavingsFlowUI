# Phase 10.7: Code Review Report

**Reviewed:** 2026-06-29T05:26:01Z
**Depth:** standard
**Files Reviewed:** 22
**Status:** issues_found

## Summary

Step 1 계좌흐름도 복원, 수입 allocation, 수동 이체, 잉여현금 자동 이체, Portfolio handoff 제거 의도를 기준으로 표준 리뷰를 수행했다. 현재 구현은 계좌흐름도 UI 자체는 노출하지만, 데이터 경로가 여러 곳에서 실제 금액과 불일치한다. 특히 수동 이체는 sanitize/snapshot에는 남아 있으나 Network/Sankey 데이터로 들어가지 않고, 잉여현금 자동 이체는 특정 투자 버킷에서 누락된다.

## Critical Issues

### CR-01: 수동 이체 규칙이 Network/Sankey 데이터에 반영되지 않음

**File:** `apps/main/modules/sankey-builder.js:378`

**Issue:** `buildMonthlySnapshot()`은 `inputs.transfers`를 `snapshot.transfers`로 넘기지만, `buildSankeyData()`는 line 378에서 빈 `transfers` 배열을 새로 만들고 수입 allocation 및 자동 잔액 맞춤만 추가한다. `sanitizeTransfers()`와 `renderTransferRulesList()`는 여전히 수동 이체 데이터를 다루지만, `render-orchestrator.js:62-77`은 `sankeyData.transfers`만 Network Map에 전달하므로 저장된 수동 이체 규칙은 계좌흐름도에서 사라진다.

**Fix:** `snapshot.transfers`를 `sourceAccountId/targetAccountId`에서 Network 형식으로 변환해 `transfers` 초기값에 포함하고, `isManual: true`를 붙인 뒤 provider/consumer 잔액 계산에도 반영한다.

```javascript
const transfers = (Array.isArray(snapshot.transfers) ? snapshot.transfers : [])
  .map((transfer) => ({
    id: transfer.id,
    source: resolveAccountId(transfer.sourceAccountId, "transfer"),
    target: resolveAccountId(transfer.targetAccountId, "transfer"),
    value: Number(transfer.amount) || 0,
    tone: "transfer",
    label: transfer.label || "계좌 이체",
    isManual: true,
  }))
  .filter((transfer) => transfer.source && transfer.target && transfer.source !== transfer.target && transfer.value > 0);
```

### CR-02: 잉여현금 자동 이체가 두 번째 이후 투자 항목에 배분되지 않음

**File:** `apps/main/modules/calculator.js:351`

**Issue:** `targetInvests.forEach()`에서 `idx = investBuckets.indexOf(bucket)`를 구한 뒤 `surplusAdds[targetSavings.length + idx]`를 사용한다. `surplusAdds`는 `targetBuckets` 기준 배열이므로, 대상 투자 버킷이 `investBuckets`의 1번 이상 인덱스에 있으면 잘못된 위치를 읽어 `undefined`가 되고 잉여현금이 버킷에 더해지지 않는다. 자동 이체를 투자계좌로 보내도 시뮬레이션 잔고가 실제보다 낮아진다.

**Fix:** `targetInvests`의 로컬 인덱스를 사용한다.

```javascript
targetInvests.forEach((bucket, localIndex) => {
  bucket.balance += surplusAdds[targetSavings.length + localIndex] || 0;
});
```

### CR-03: 수입 allocation 합계가 수입 금액과 달라도 그대로 Sankey에 사용됨

**File:** `apps/main/modules/input-sanitizer.js:166`

**Issue:** `sanitizeIncomeItems()`는 `income.amount`와 `income.allocations[].amount`의 합계를 맞추지 않는다. 이후 `sankey-builder.js:318-347`은 수입 노드에서 총수입으로는 `src.value`를 보내고, 총수입에서 계좌로는 allocation 합계를 보낸다. allocation 합계가 수입보다 크거나 작으면 `총수입` 노드의 입출금이 불일치하고, 계좌흐름도/요약의 입금 금액이 실제 월수입과 달라진다.

**Fix:** sanitize 단계에서 allocation 합계를 `safeAmount`에 맞게 스케일하거나, 합계 불일치 시 단일 기본 allocation으로 복구한다. 최소한 `buildSankeyData()`에서 `src.value`와 allocation 합계 차이를 보정하는 링크를 생성해야 한다.

### CR-04: Portfolio 저장 데이터가 속성 컨텍스트에서 이스케이프되지 않아 저장형 XSS 경로가 남음

**File:** `apps/portfolio/modules/dom.js:58`

**Issue:** 포트폴리오 목록은 `innerHTML`로 렌더링하면서 `data-id="${p.id}"`, `data-period`에 해당하는 `${p.period}`를 그대로 삽입한다. 이름은 escape하지만 id/period는 저장소에서 로드되는 값이므로, import/restore 또는 변조된 localStorage가 들어오면 속성 탈출 기반 스크립트 삽입이 가능하다. 같은 패턴이 creator/modal asset row의 `data-asset-id="${as.id}"`, `data-id="${as.id}"`에도 반복된다.

**Fix:** 모든 속성 삽입값에 `IsfUtils.escapeHtml()`을 적용하거나, `document.createElement()`와 `dataset`/`textContent`로 DOM을 구성한다.

## Warnings

### WR-01: 테스트가 현재 사용자 의도와 반대로 수동 이체 제거를 고정함

**File:** `tests/step1.spec.ts:1125`

**Issue:** 테스트명이 `removes manual transfer settings...`이고, `#transferEditorSection` 부재 및 `legacyManual === false`를 기대한다. 현재 리뷰 요청의 기준은 Step 1 계좌흐름도 기능 복원과 수동 이체 규칙 확인이므로, 이 테스트는 복원 요구를 검증하지 못하고 오히려 수동 이체 재연결을 막는 회귀 테스트가 된다.

**Fix:** 수동 이체를 최종적으로 제거할 의도라면 소스의 `transfers`/수동 이체 잔여 핸들러를 제거한다. 복원할 의도라면 테스트를 `legacyManual === true`, Network edge 표시, 저장/재렌더 후 유지까지 검증하도록 바꾼다.

### WR-02: ADR이 현재 HEAD의 Portfolio handoff 제거 및 Step 1 계좌흐름도 복원과 충돌함

**File:** `docs/adr/0002-account-flow-belongs-to-portfolio-boundary.md:3`

**Issue:** ADR은 Step 1이 계좌 선택, allocation editor, network-map controls를 복원하면 안 되고 `accountFlowHandoff`를 Portfolio sidecar로 유지해야 한다고 명시한다. 현재 HEAD는 Step 1에 `계좌흐름도` 탭을 복원했고, 리뷰 요청도 Portfolio handoff 제거를 요구한다. 문서가 반대 정책을 유지하면 후속 구현/리뷰가 잘못된 기준으로 회귀할 가능성이 높다.

**Fix:** ADR을 superseded 처리하고, Step 1이 소유하는 계좌흐름도/수입 allocation/잉여현금 자동 이체 범위와 Portfolio가 더 이상 handoff 대상이 아님을 새 ADR 또는 개정 ADR로 명시한다.

## Info

### IN-01: Portfolio Step1Connector가 더 이상 연결되지 않는 죽은 경로로 보임

**File:** `apps/portfolio/modules/step1-connector.js:7`

**Issue:** `Step1Connector` export는 `apps/portfolio` 내 어디에서도 import되지 않는다. Portfolio handoff 제거가 맞다면 이 파일은 남아 있는 혼동 요소이고, handoff 유지가 맞다면 기능이 실제 Portfolio UI에 연결되지 않은 상태다.

**Fix:** handoff 제거 정책이면 파일과 관련 테스트/문서를 제거한다. 유지 정책이면 `app.js` 또는 Portfolio state 흐름에서 명시적으로 호출하고 E2E로 검증한다.
