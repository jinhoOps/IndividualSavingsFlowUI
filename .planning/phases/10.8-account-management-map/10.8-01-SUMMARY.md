### Auto-fixed Issues

**1. [Rule 1 - Bug] Corrected connector source priority for current Main imports**
- **Found during:** Task 2 targeted Playwright verification.
- **Issue:** The first connector implementation could prefer an older IndexedDB latest snapshot over the active `isf-rebuild-v1` seed, producing stale relationship counts.
- **Fix:** `resolveLatestMainInputs()` now sanitizes and returns bridge/local `isf-rebuild-v1` data first, then falls back to IndexedDB snapshot only when active local data is absent.
- **Files modified:** `apps/account-map/modules/step1-connector.js`
- **Verification:** `npx playwright test tests/account-map.spec.ts --reporter=list` passed.
- **Committed in:** `6817028`

**2. [Rule 1 - Test Contract] Aligned Account Map test seed with Main allocation sanitizer contract**
- **Found during:** Task 2 targeted Playwright verification.
- **Issue:** The route test expected split income allocations while the seed omitted `splitIncomeAccounts: true`, so Main sanitizer correctly collapsed allocations to a single account.
- **Fix:** Added `splitIncomeAccounts: true` to the seed and set the route relationship expectation to 7 relationships.
- **Files modified:** `tests/account-map.spec.ts`
- **Verification:** `npx playwright test tests/account-map.spec.ts --reporter=list` passed.
- **Committed in:** `6817028`
