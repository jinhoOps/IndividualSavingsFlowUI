## Overview

Create the dedicated Account Map app foundation for Phase 10.8. This plan establishes the new multi-page Vite route, page-owned draft storage, a bridge-aware Main-data connector, and deterministic draft generation from sanitized Step 1 data.

Purpose: make Account Map a separate destination per D-06 and D-08, while preserving Main as the data source and avoiding an empty first-run setup per D-09.

Output: a loadable `/apps/account-map/` page with import-ready draft state and automated tests for draft scope.

## Scope

In scope:

- Add `apps/account-map/` as a standalone route following existing Vite multi-page conventions.
- Add Account Map page-owned storage using a new key such as `isf-account-map-v1`.
- Read latest Main/Step 1 data through the storage compatibility bridge or the same fallback pattern used by Step 2.
- Generate draft accounts, relationships, and candidates from Main data:
  - accounts;
  - income allocation/deposit relationships;
  - manual transfer relationships;
  - savings transfer relationships;
  - investment/brokerage transfer relationships;
  - fixed-expense candidates.
- Mark obvious recurring-payment keyword fixed expenses as recommended when implemented per D-13; at minimum, all fixed-expense candidates must be listed for review per D-12.
- Exclude variable expenses and one-off spending from accepted relationships by default per D-02 and D-11.

Out of scope:

- No Portfolio route/module changes in this plan.
- No relationship filter tabs; store `type` metadata only per D-05.
- No live bank/card scraping or live integrations.
- No writes back into Main data; this plan creates Account Map-owned draft state only.
