## Map Relationship Scope

| Option | Description | Selected |
|--------|-------------|----------|
| Automatic transfers plus card/utility links | Show salary/living-account transfers, card payment links, and utility/telecom/insurance links. | |
| Automatic transfers plus card/utility links plus savings/investment transfers | Include the above plus savings, deposit, and brokerage/investment transfers. | ✓ |
| All Main expense/savings/investment items as account relationships | Pull nearly all Main items into account flow. Richer but risks becoming another budget screen. | |

**User's choice:** Include automatic transfers, card/utility links, and savings/investment transfers.
**Notes:** General spending categories should not appear as primary account-map relationships.

## Relationship Detail Display

| Option | Description | Selected |
|--------|-------------|----------|
| Show accounts and external destinations as nodes | Card and utility destinations appear as nodes. | |
| Show only accounts as nodes and keep details inside account cards | Simpler map, weaker relationship-map feel. | |
| Show high-level relationships in map and details in a side panel | Default map remains simple; selecting an account reveals card/utility details. | ✓ |

**User's choice:** High-level map first, details on selection.
**Notes:** Amounts are hidden by default and shown on selection. Relationship types use colors and short labels. Filter tabs are deferred as a future focus feature.

## Screen Location

| Option | Description | Selected |
|--------|-------------|----------|
| Dedicated section inside Main | Easy access but risks making Main complex again. | |
| Portfolio account-management tab | Reuses Portfolio route but mixes account map with stock/portfolio management. | |
| Separate Account Map page | Clean dedicated destination, but adds route/entry/storage/tests. | ✓ |
| Main mini-map plus separate detailed page | Main provides easy entry while full map lives on its own page. | ✓ |

**User's choice:** Separate Account Map page plus Main mini-map/entry.
**Notes:** User selected “4+3”: the destination is a dedicated page; Main only provides a lightweight entry signal.

## Input Model

| Option | Description | Selected |
|--------|-------------|----------|
| Create accounts first | User registers accounts, then connects relationships. | |
| Create relationships first | User adds real relationships and accounts are created as needed. | |
| Start from role presets | Begin with salary/living/savings/investment account roles. | |
| Generate a draft from Main data and then edit | Main data creates the initial draft; user edits relationships after. | ✓ |

**User's choice:** Generate an initial draft from Main data, then edit by adding/modifying relationships.
**Notes:** The user does not want an empty setup flow. Relationship-first editing after draft generation is the preferred management model.

---

## Main Data Import

| Option | Description | Selected |
|--------|-------------|----------|
| Income + savings + investment only | Clean but leaves living/card/payment side manual. | |
| Income + savings + investment + fixed expenses | Pulls likely recurring account relationships into draft. | ✓ |
| Income + savings + investment + fixed expenses + recurring variable expenses | Richer, but risks mixing budget and account-map concepts. | |
| User chooses imported items | Accurate but more setup work. | ✓ |

**User's choice:** Use income/savings/investment/fixed expenses as the default import scope, with user selection for candidates.
**Notes:** Variable and one-off expenses are excluded by default.

---

## Draft Generation Rules

| Option | Description | Selected |
|--------|-------------|----------|
| Name/group keyword based | Use recurring-payment keywords to detect likely relationships. | |
| User-visible group based | Treat all Main fixed expenses as candidates. | ✓ |
| Keyword auto-include plus remaining fixed expenses as candidates | Balanced but potentially harder to implement. | ✓ |
| All fixed expenses require selection | Safest but more manual. | |

**User's choice:** Fixed expenses should all become Account Map candidates. If feasible, keywords can mark strong recommendations and ambiguous candidates.
**Notes:** Minimum requirement is fixed-expense candidate listing. Keyword classification is optional enhancement.

## First Screen Layout

| Option | Description | Selected |
