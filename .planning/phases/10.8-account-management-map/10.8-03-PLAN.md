## Overview

Add the lightweight Main entry and shared navigation for the new Account Map destination while preserving existing Main and Portfolio boundaries. Main should expose a small summary/mini-map/status entry that routes to the Account Map page, not the full relationship editor.

Purpose: satisfy D-07 and D-16 without reintroducing a complex account-flow editor into Main or falling back to the superseded Portfolio-only boundary.

Output: Account Map appears in shared navigation and Main has a compact entry point with targeted regression coverage.

## Scope

In scope:

- Add Account Map to the shared app launcher with a distinct label and active state.
- Add a compact Main summary/mini-map entry that links to `../account-map/`.
- Derive the Main entry summary from existing visible/sanitized Main data where available: account count, relationship/candidate rough count, and confirmation-needed count or generic status.
- Keep Main scan-only for this feature; no Account Map relationship editor inside Main.
- Add regressions proving Main account-flow compatibility fields and restored behavior are not removed as collateral damage.

Out of scope:

- No Portfolio changes.
- No old Portfolio-only account-flow boundary.
- No removal of restored Main account-flow behavior in this phase.
- No live bank/card scraping.
- No relationship filter tabs.
