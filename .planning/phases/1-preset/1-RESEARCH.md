# Phase 1: 프리셋 템플릿 로드 및 자동 시각화 - Research

## 1. Domain & Codebase Analysis

### 현재 아키텍처 및 상태 관리
- **위치:** `apps/step1` 디렉터리에 애플리케이션 핵심 로직이 구현되어 있습니다.
- **모듈화:** `state.js`, `calculator.js`, `sankey-renderer.js`, `dom.js` 등으로 로직이 분리되어 있습니다. 
- **데이터 구조:** `state.js` 내에 자산, 지출, 저축 등을 담는 데이터 모델이 존재하며, 단위는 '원'으로 관리되고 UI 표시 시 '만원'으로 변환하는 구조를 따릅니다 (`IsfUtils` 활용).

### 추가해야 할 기능 요소
1. **프리셋 UI (HTML/CSS):** 
   - `apps/step1/index.html`에 연봉 드롭다운 및 투자 성향 버튼(안정형/중립형/적극형)을 포함하는 프리셋 영역 추가.
   - `apps/step1/styles.css`에 BEM 표기법을 준수하여 프리셋 UI 스타일링.
2. **프리셋 데이터 정의:**
   - 연봉 수준별 기본 세후 수입 산출, 성향별 분배 비율(생활비, 저축, 투자 등) 정의. `modules/preset-templates.js` 같은 새 모듈 추가 혹은 `constants.js` 확장 필요.
3. **사용자 확인 (Confirm 팝업):**
   - 기존 데이터(`state.js` 내 값이 기본값이 아닌 경우)가 있을 때 덮어쓰기 여부를 묻는 네이티브 `confirm` 또는 커스텀 모달 구현.
4. **상태 업데이트 로직:**
   - 선택된 프리셋을 기반으로 동적 수식을 통해 각 항목(Income, Expense, Saving, Investment) 계산.
   - `calculator.js` 또는 `state-helpers.js`를 이용해 계산된 값을 `state`에 덮어쓰기.
   - `sankey-renderer.js` 트리거를 통한 즉각적 시각화 업데이트 (기존 `markDirty` 및 렌더링 사이클 활용).

## 2. Dependencies
- **Vanilla JS 제약:** 외부 프레임워크나 빌드 도구가 없으므로, ES 모듈 방식으로 브라우저에서 직접 로드되도록 설정.
- **물리적 무결성 (CSS):** 기존 미디어 쿼리가 손상되지 않도록 파일 최하단에 반응형 스타일이 안전하게 유지되어야 함.
- **단위 일치 (원/만원):** 연봉 프리셋(예: 3천만 원)을 내부 상태로 로드할 때는 '원' 단위(30,000,000)로 변환하여 저장할 것.

## 3. Potential Landmines
- **상태 동기화 이슈:** 데이터를 `state.js`에 강제로 덮어씌운 뒤 DOM 요소(입력 필드)들이 이전 값을 계속 보여주면 안 됨. 상태 업데이트 시 모든 연관 DOM 입력 필드의 값을 최신 `state` 값으로 갱신하는 헬퍼 함수가 필요.
- **기존 데이터 유실 위험:** 프리셋 버튼 클릭 실수로 사용자의 기존 데이터가 날아갈 수 있으므로, 상태가 기본값(`isPristine` 등)인지 확인하는 로직이 반드시 포함되어야 함.

## 4. What we need to PLAN
1. `index.html`에 프리셋 영역 설계 및 배치.
2. 프리셋 계산식 및 상수 객체 (연봉별 수입, 성향별 비율) 구성 (`preset-data.js`).
3. 이벤트 리스너 추가: 드롭다운 변경 및 버튼 클릭 시 적용 로직 (`app.js` 혹은 이벤트 바인딩 모듈).
4. `updateStateFromPreset()` 같은 상태 업데이트 함수 구현 및 UI/Sankey 연동 로직.
5. `confirm` 팝업을 통한 데이터 덮어쓰기 검증 로직 구현.
