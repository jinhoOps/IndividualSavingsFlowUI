# Phase 03 Plan 01 Execution Summary

## 🎯 실행 결과 요약
본 단계에서는 다중 계좌 데이터 모델 정교화 및 프리미엄 UX 개편 계획인 `03-01-PLAN.md`를 차례대로 수행하였습니다. Sankey 다이어그램의 선 꼬임과 가독성 문제를 해소하기 위해 내부 이체 데이터를 다이어그램의 선에서 원천 분리하고, 이를 대체할 전용 '계좌 간 자동 이체 흐름' 보드를 하단에 신설하였습니다. 또한 연간 이자/배당 금융소득의 누적 금액을 실시간 계산하여 임계 구간(1,900만 원, 3,400만 원)을 넘을 시 카드 아웃라인 링 및 뱃지를 통해 경고하는 금융 종합소득세 인디케이터를 연동하였습니다.

## 🛠️ 태스크별 수행 내역 및 정합성 검증

### Task 1: index.html 마크업 개편 및 dom.js DOM 매핑 추가
- **구현 내용**: `index.html` 내 `#sankeyLegend` 하단에 이체 보드 영역 `#accountTransferBoard` 및 카드 피드가 동적으로 렌더링될 `#transferCardList` 컨테이너를 추가하였습니다.
- **검증**: `dom.js`에 새로 추가된 DOM 셀렉터 프로퍼티인 `accountTransferBoard`와 `transferCardList`가 정확하게 매핑되었음을 확인했습니다.

### Task 2: sankey-builder.js 로직 수정
- **구현 내용**: `buildSankeyData` 함수를 수정하여 `tone === "transfer"`인 내부 이체 링크가 `links` 배열에 들어가지 않도록 제외하고, 이를 별도 `transfers` 배열에 수집해 반환하도록 아키텍처를 단순화하였습니다.
- **검증**: 이체 선이 빠지더라도 계좌 노드의 시각적 두께(value)가 유실되지 않도록 `[...links, ...transfers]`를 함께 순회하여 계좌 유입 금액을 누적 연산하도록 보정하였습니다.

### Task 3: calculator.js에 금융소득 및 과세 경고 계산 기능 추가
- **구현 내용**: `calculateAccountFinancialIncomes(inputs)` 함수를 신설하였습니다. 저축 버킷(시작 잔고 * 연이율 / 100) 및 투자 버킷(시작 잔고 * 2% 고정 배당)의 연간 이자/배당을 계좌별로 분류하여 합산하고, 전체 금융소득이 1,900만 원 초과 시 `warn`("금융소득 종합과세 주의"), 3,400만 원 초과 시 `crit`("금융소득 종합과세 대상 (한도 초과)") 등급을 적용해 금융소득을 유발하는 모든 관련 계좌의 경고 상태 맵(`warnings`)을 반환합니다.
- **검증**: `IsfUtils`의 화폐 단위 변환 및 경고 등급 조회 로직을 완전 재사용하여 단위 오차가 없도록 정합성을 수호하였습니다.

### Task 4: list-renderer.js 컴포넌트 렌더러 구현
- **구현 내용**: 이체 내역(`transfers`)과 계좌명 정보(`accounts`)를 전달받아 정갈한 이체 현황 카드(`.transfer-card`)로 렌더링하는 `renderTransferBoard`를 구현하였습니다. 또한, 계좌 렌더러(`renderAccountItemHtml`)를 확장하여 경고 상태(`opts.warnings`)가 전달되었을 시 테두리 스타일(`account-row--warn`, `account-row--crit`) 및 법적 카피라이트 규격을 준수하는 경고 뱃지를 점등하도록 구성하였습니다.
- **검증**: `renderItemList` 함수 호출 시 2중 방어(Double Protection) 기법을 도입하여, 컨트롤러가 누락된 옵션 상태를 전달하더라도 렌더러 단에서 최신 상태를 실시간 역계산하여 경고 링과 뱃지가 깨짐 없이 점등되도록 fallback을 강화하였습니다.

### Task 5: app.js 컨트롤러 연동 및 UI 라이프사이클 통합
- **구현 내용**: `app.js` 내 `renderAll()`의 생키 렌더링 파이프라인에서 `calculateAccountFinancialIncomes`를 연동하여 실시간 계좌 경고 정보(`warnings`)와 내부 이체 배열(`transfers`)을 취득한 후, 각각 `refreshInputsPanel(..., warnings)`과 `listRenderer.renderTransferBoard`로 넘겨 화면에 그리도록 생명주기를 유기적으로 통합하였습니다.
- **검증**: 사용자가 데이터를 편집하거나 적용(`applyChanges` 등)할 때에도 변경 사항에 맞게 계좌의 종합소득 과세 등급과 내부 이체 흐름이 부드럽게 재계산되어 실시간 동기화됨을 확인하였습니다.

### Task 6: styles.css 테마 및 반응형 레이아웃, 상세 인터랙션 모션 구현
- **구현 내용**: 
  - **이체 보드**: PC 해상도(>= 768px) 기준 3열 그리드로 넓게 배치하고, 모바일 해상도(<= 760px) 기준 수직 피드 스택으로 변환하며 스크롤 관성을 보완하였습니다. 모바일에서의 카드 패딩은 12px 콤팩트 규격을 지켰습니다.
  - **프리미엄 모션**: 이체 카드 마우스 호버 시 Sunset Orange 테두리 변환 효과와 함께 Y축 -2px 수직 부유 효과를 부여하고, 이체 흐름 방향 화살표 기호(`➔`)에는 가로 방향(X축) 바운스 키프레임 애니메이션을 적용해 생동감을 주었습니다.
  - **경고 스타일**: 경고 및 한도 초과 등급에 맞춰 Yellow/Red 링 아웃라인과 `box-shadow` 글로우 및 점등 애니메이션을 매끄럽게 연출했습니다.
  - **물리적 무결성**: 파일 크기와 하단의 기존 반응형 미디어 쿼리를 전혀 훼손하지 않고 정교한 CSS 추가만을 수행하였습니다.

## 📈 검증 완료 정의 (DoD)
1. **Sankey 단순화**: 다이어그램 내 계좌 간 꼬임 현상을 유발하던 내부 이체(tone: 'transfer') 링크가 제외되어 시각적으로 단순화되었습니다.
2. **이체 흐름 보드**: 생키 하단에 '계좌 간 자동 이체 흐름' 보드가 신설되었고 PC 3열 그리드 및 모바일 수직 피드 스택이 완벽히 적용되었습니다.
3. **금융소득 과세 인디케이터**: 연간 누적 금융소득이 1,900만 원 초과 시 경고 링/뱃지("금융소득 종합과세 주의"), 3,400만 원 초과 시 위기 링/뱃지("금융소득 종합과세 대상 (한도 초과)")가 타깃 계좌에 실시간 점등됩니다.
4. **인터랙션 피드백**: 부드러운 수직 부유 트랜지션 및 화살표 가로 바운스 키프레임 효과가 정상 동작합니다.
5. **빌드 무결성**: 전체 CSS 구조 및 ES 모듈 임포트 구조가 보존되어 정적 컴파일 오류를 유발하지 않습니다.
