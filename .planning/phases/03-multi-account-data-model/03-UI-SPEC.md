## Visual Hierarchy & Focal Points

### 1. 주요 화면별 포커스 포인트 (Focal Point)
- **계좌 간 이체 현황 보드 (D-02)**:
  - **1순위 (Primary Focal Point)**: 이체 방향을 가리키는 정밀한 화살표 아이콘(`➔`) 및 이체 금액 배지 (`.badge-transfer`).
  - **2순위 (Secondary Focal Point)**: 출발 계좌명과 도착 계좌명.
- **금융종합소득 및 안전 한도 경고 (D-03)**:
  - **1순위**: 이자/배당 소득이 기준치를 초과할 때 점등되는 경고 뱃지(`.status-badge--warn`, `.status-badge--crit`) 및 붉은 아웃라인 링.

### 2. 요소별 시각적 위계 (Visual Hierarchy)
- **Sankey Diagram의 간소화 (D-01)**:
  - 복잡한 내부 이체(transfer) 선을 차트에서 완전 제외함으로써, 수입원 ➔ 계좌 ➔ 지출/저축/투자로 이어지는 흐름만 굵고 깨끗하게 표현합니다.
  - 이를 통해 생키 차트는 지출 총합과 분포(60-30-10) 위계를 파악하는 데에만 집중시킵니다.
- **계좌 간 이체 현황 보드의 구조화 (D-02)**:
  - 모바일에서는 1열 세로 피드 스택 형태로 정렬되어 한 눈에 위에서 아래로 이체 정합성을 보게 하고, PC 와이드 뷰에서는 3열 그리드로 넓게 배치하여 시각적 인지 부하를 분산시킵니다.

## UX & Interaction Contract (UX 스펙 추가)

### 1. Sankey Diagram Simplification (생키 차트 역할 정의)
- **목적**: 데이터 꼬임 현상과 가로/세로 화면 스크롤 문제를 근본적으로 해결하기 위해, 생키 차트에서는 계좌 간 N:N 이체 링크를 완전히 제거합니다.
- **표시 영역**: `수입원 ➔ 계좌 ➔ 지출/저축/투자 대분류`로 구성된 3단(Column 0, 1, 2) 흐름만 명확하게 렌더링합니다.

### 2. Account Transfer Board (계좌 간 이체 현황 보드) 인터랙션
- **모바일 스택 뷰**: 모바일 화면(<= 760px)에서 계좌 이체 보드의 카드 리스트는 세로 1열 피드 형태로 렌더링되며, 터치 조작 감도를 높이기 위해 `-webkit-overflow-scrolling: touch` 속성을 부여하여 관성 스크롤을 활성화합니다.
- **PC 그리드 뷰**: PC 화면(>= 768px)에서는 3열 그리드로 와이드하게 정렬되어 한 눈에 다중 계좌 데이터 흐름을 대조하기 용이하게 구성합니다.
- **인터랙티브 피드백**: 
  - 각 이체 카드(`.transfer-card`)는 마우스 호버(Hover) 시 테두리 색상이 `rgba(16, 34, 32, 0.12)`에서 `rgba(234, 91, 42, 0.3)` (Sunset Orange 계열)로 부드럽게 강조되며, `transform: translateY(-2px)` 트랜지션(`transition: all 0.2s ease-in-out`)을 제공합니다.
  - 이체 방향 화살표(`➔`)에는 미세한 가로 애니메이션 바운스 효과를 얹어 흐름의 방향성을 직관적으로 드러냅니다.

### 3. Safety Margin Warning Indicator (금융소득 경고 인디케이터) UX
- **계좌 카드 연동**: 연간 이자/배당 금융소득의 합산 총액에 근거하여, 금융소득을 유발하는 관련 저축/투자 계좌 카드(`.account-row`)의 테두리를 강조 점등합니다.
  - **Soft Warn (1,900만 원 초과)**: 테두리 컬러 `#f59e0b` (Yellow) 변경 및 `box-shadow: 0 0 0 2px rgba(245, 158, 11, 0.15)` 적용.
  - **Critical Warn (3,400만 원 초과)**: 테두리 컬러 `#dc2626` (Red) 변경 및 `box-shadow: 0 0 0 2px rgba(220, 38, 38, 0.15)` 적용.
- **경고 상태 애니메이션**: 경고 뱃지(`.status-badge`)가 렌더링되거나 숨겨질 때 `opacity`와 `transform` 트랜지션을 이용해 페이드인/아웃 애니메이션을 연출하여 끊김 없이 자연스럽게 사용자 시선을 유도합니다.
