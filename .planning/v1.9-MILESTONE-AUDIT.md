## Cross-Phase Issues

### HH-02 Is Data-Only In Current UI

Phase 10 added and verified one-income/dual-income household controls. Phase 10.5 then removed the visible household/couple UI as an explicit product decision, while leaving the underlying sanitizer and data model intact. That means HH-02 is no longer satisfiable as a user-facing requirement in the current app.

This is not a Phase 10.5 implementation defect. It is a scope conflict between the original v1.9 milestone and the later Phase 10.5 decision to defer couple/household-status UI to a future dedicated spec.

### Phase 11 Capture Is Not Wired To Actual Spending

Existing parser/Smart Add code is not enough for the Phase 11 contract. The current opener is not present in the DOM, and the existing apply path updates planned expense amounts or creates a new expense item instead of updating variable expense `actualSpent`.

### Phase 12 Merge Lacks Preview And Conflict Review

Existing DataHub merge behavior decodes partner data and commits concatenated item arrays directly. It does not provide a preview, shared-node merge policy, or conflict warning review step.

### Phase 13 Comparison Is Disabled

The comparison surface is present as disabled/coming-soon UI. Snapshot selection, chart rendering, labels/tooltips, mobile behavior, and empty state are not verified.

### Phase 14 Affordability Planner Is Missing

No affordability planner implementation surface was found for DSR/LTV/cash/rate/term assumptions, result table, scenarios, or plain-language classification.

## Positive Wiring

- The default `재무설정 상세` CTA opens the integrated financial modal.
- Financial summary cards route into the same integrated modal path.
- Variable living-expense rows reuse Phase 10 household budget helpers for target, actual, remaining, status, and projection.

## Recommendation

Do not archive v1.9 as complete.

The practical next move, based on the user's current direction, is to treat Phase 10.5 as the completed stopping point and defer the rest of v1.9. Before continuing the milestone, reconcile HH-02:
