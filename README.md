# IndividualSavings Flow UIUX

개인 재무 흐름을 입력하고, 그 흐름이 장기 자산 계획과 포트폴리오 실행 계획으로 어떻게 이어지는지 확인하는 로컬 우선 웹 앱입니다.

ISF는 하나의 거대한 금융 앱이라기보다, 세 개의 연결된 도구로 구성된 계획 워크스페이스입니다.

- **Main**: 월간 현금흐름과 계좌 흐름을 정리합니다.
- **Simulation**: Main의 월 투자 여력을 바탕으로 투자 전략별 장기 결과를 비교합니다.
- **Portfolio**: 실제 적립식 포트폴리오 구성을 만들고 저장합니다.

배포 페이지: https://jinhoops.github.io/IndividualSavingsFlowUI/

## 프로젝트 목표

ISF의 목표는 사용자가 복잡한 재무 모델을 직접 다루지 않아도 다음 질문에 답할 수 있게 하는 것입니다.

- 내 월 수입은 어디로 흘러가고 있는가?
- 생활비, 저축, 투자 비중은 지금 계획대로 괜찮은가?
- 남는 현금흐름을 투자로 돌리면 장기적으로 어떤 차이가 나는가?
- 선택한 전략을 실제 포트폴리오로 옮기려면 어떤 구성이 필요한가?

서버 계정이나 은행 연동 없이, 사용자가 직접 입력한 데이터를 브라우저 안에서 계산하고 시각화하는 방향을 유지합니다.

## 앱 구성

### Main

Main은 프로젝트의 시작점입니다. 월 수입, 생활비, 저축, 투자, 계좌를 입력하고 Sankey Diagram과 Network Map으로 흐름을 확인합니다.

현재 Main의 핵심 UX는 요약 먼저, 상세 편집은 모달에서 처리하는 구조입니다. `재무설정 상세`에서는 수입, 생활비, 투자, 저축, 자동 저축 결과를 한 흐름으로 확인하고 수정합니다. 변경사항은 바로 저장되지 않고, 모달 내부 펜딩바에서 `적용`해야 반영됩니다.

Main에서 다루는 주요 내용:

- 월간 현금흐름 요약
- 수입, 생활비, 저축, 투자 항목 관리
- 계좌 기반 Sankey 흐름
- 계좌 관계 Network Map
- 장기 Projection Table
- 프리셋 기반 초기 설정
- 데이터 허브를 통한 백업, 복원, 공유

### Simulation

Simulation은 Main에서 만든 현금흐름을 장기 투자 전략 비교로 확장합니다.

지수 성장, 배당 성장, 커버드콜/월 현금흐름처럼 성격이 다른 전략을 비교하고, 최종 자산과 월 현금흐름 관점에서 어떤 tradeoff가 있는지 보여줍니다. Step 1의 투자 여력을 가져오되, Simulation에서 별도로 수정하고 저장할 수 있습니다.

Simulation에서 다루는 주요 내용:

- 투자 전략 카드 선택
- 전략별 장기 자산 비교
- 월 현금흐름 비교
- 배당/분배금 기반 현금흐름 모델
- 저장된 시뮬레이션 불러오기와 삭제
- 데이터 허브 연동

### Portfolio

Portfolio는 계획을 실제 적립식 포트폴리오 형태로 옮기는 화면입니다.

포트폴리오 이름, 적립 주기, 종목/자산별 금액을 입력하고, 구성 비중과 총액을 확인한 뒤 저장합니다. 생성된 포트폴리오는 다시 열어 수정할 수 있습니다.

Portfolio에서 다루는 주요 내용:

- 적립식 포트폴리오 생성
- 종목/자산별 금액 입력
- 총 매수 금액과 비중 확인
- 포트폴리오 상세 수정
- 포트폴리오 삭제
- 로컬 저장

## 공유 인프라

세 앱은 공통 인프라를 공유합니다.

- 공통 헤더와 앱 런처
- 데이터 허브 모달
- 로컬 저장소와 IndexedDB 백업
- JSON 내보내기/가져오기
- ISF CODE 기반 공유
- PWA 매니페스트와 서비스워커
- 공통 디자인 토큰과 버튼/패널 스타일

데이터는 기본적으로 브라우저 로컬에 저장됩니다. 사용자가 JSON 또는 ISF CODE를 직접 내보내는 경우에만 외부로 이동합니다.

## 제품 원칙

- **요약 먼저**: 처음 보는 화면은 세부 입력 폼보다 현재 상태를 읽는 화면이어야 합니다.
- **명시적 저장**: 큰 편집 흐름은 사용자가 적용하기 전까지 draft로 유지합니다.
- **로컬 우선**: 서버 계정 없이 브라우저 저장소와 백업으로 동작합니다.
- **한국어 금액 UX**: 사용자는 만 원/억 원 단위로 읽고, 내부 계산은 원 단위로 유지합니다.
- **시각화 중심**: 숫자 표만 보여주지 않고 Sankey, Network, 전략 비교, 포트폴리오 비중으로 흐름을 드러냅니다.
- **단계별 확장**: 현금흐름, 전략 비교, 포트폴리오 실행을 분리하되 데이터 흐름은 연결합니다.

## 실행하기

필요 조건:

- Node.js 20 이상 권장
- npm

설치:

```bash
npm install
```

개발 서버:

```bash
npm run dev
```

타입 체크:

```bash
npm run check
```

전체 E2E 테스트:

```bash
npm run test:e2e
```

빌드:

```bash
npm run build
```

## 개발 구조

이 저장소는 Vite 기반 정적 웹 앱입니다. 현재는 바닐라 ES 모듈 중심의 앱 구조와 일부 React 의존성이 함께 있습니다. React는 점진적 전환과 향후 UI 확장을 위한 기반으로 남아 있습니다.

큰 책임 경계는 다음과 같습니다.

- **앱 부팅**: 초기 컨트롤러 생성, 이벤트 바인딩, 첫 렌더
- **입력/상태 관리**: 사용자 입력 정규화, draft, pending, 저장 경계
- **계산**: 월간 현금흐름, 계좌 흐름, 장기 projection, 투자 전략 비교, 포트폴리오 비중
- **렌더링**: 요약 카드, Sankey, Network, 전략 비교 카드, 포트폴리오 카드
- **저장/공유**: 로컬 저장, IndexedDB 백업, JSON, ISF CODE
- **공통 UI**: 헤더, 데이터 허브, 테마, PWA 관련 요소

## 검증 기준

작업 전후 최소 확인:

```bash
npm run check
```

사용자 흐름을 건드렸다면 E2E 테스트를 실행합니다.

```bash
npm run test:e2e -- --reporter=list
```

Main의 재무설정 상세 모달만 수정했다면 빠른 회귀로 해당 그룹을 먼저 확인할 수 있습니다.

```bash
npm run test:e2e -- -g "Phase 10.6" --reporter=list
```

## 현재 로드맵

현재 개발 흐름은 Main의 가계/현금흐름 기반을 안정화한 뒤, 다음 기능으로 확장하는 단계입니다.

- 지출 알림 텍스트를 붙여 넣어 실제 지출을 반영하는 capture 흐름
- 공유된 Main 데이터를 병합하는 household merge
- 저장된 과거 스냅샷과 현재 지출 비교
- 부동산 구매 가능성 추정
- 이후 Simulation과 Portfolio로 이어지는 데이터 연결 강화

아직 기본 화면에 들어오지 않은 기능은 각 페이즈에서 별도 UX 검증 후 반영합니다.

## 데이터와 주의사항

ISF는 금융기관 연동, 실시간 시세 연동, 법적 금융 자문을 제공하지 않습니다. 모든 결과는 사용자가 입력한 가정에 기반한 계획용 추정입니다.

대출, 세금, 투자, 부동산 의사결정에는 실제 금융기관 조건과 전문가 검토가 필요합니다.
