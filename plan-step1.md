# Step1: 나의 가계 흐름 - 개발 계획

기준 시점: `2026-03-26`, 기준 커밋: `c81a98f`

## 1) 현재 상태 요약
- 입력/계산/시뮬레이션: 수입/생활비/저축/투자 편집, 항목 그룹화, 만기 해지월, 10년 시뮬레이션까지 동작 중.
- 시각화: Sankey 금액/비율 전환, 정렬 기준 전환, 그룹 depth(상위->그룹->항목) 반영 완료.
- 저장/공유: `sid` 포인터 기반 공유 + `#s` 압축 해시 fallback + 로컬 기본 저장 경로가 정리됨.
- 백업: IndexedDB 자동 백업(12시간/최대 60개), 수동 백업 제한/안내, 레거시 localStorage 마이그레이션 반영.
- PWA: SW 캐시, 설치/업데이트 토스트, standalone 버전 체크(포그라운드 복귀 + 1일 1회 + 수동 체크) 적용.

## 2) 기술 부채 및 UX 개선 후보
- `S1-TD1` 코드 구조: `apps/step1/app.js` 단일 파일 비대화로 회귀 영향 범위 파악이 어려움.
- `S1-TD2` 테스트 공백: 공유(`sid/#s`), 백업 복원, View 모드 분기의 회귀 검증 자동화가 없음.
- `S1-UX1` 모바일 Sankey 가독성: 가로 모드/작은 화면에서 전체 흐름을 한 화면에 보기 어렵다는 사용자 이슈가 남아있음(TODO 미완료).
- `S1-UX2` 목돈성 상품 모델: 자유적립식 적금/파킹통장 같은 "잔액 중심+가변 납입" 성격을 현재 월납입 모델이 충분히 표현하지 못함(TODO 보류).
- `S1-UX3` 오류 피드백: 공유 복원 실패, 백업 충돌, 잘못된 입력값에 대한 인라인 안내가 구간별로 일관되지 않음.

## 3) 데이터/공유/백업/PWA 안정화 체크리스트
### 데이터 무결성
- [ ] 항목 추가/삭제/정렬/그룹 변경 후 `apply -> reload` 시 동일 상태 유지 검증
- [ ] 만기 해지월 적용 시 만기 이후 납입 중단 + 현금 환원 계산이 리로드 후 동일한지 검증
- [ ] 샘플/초기화/View 모드 진입 경로에서 로컬 원본 데이터 비파괴 보장

### 공유
- [ ] `?view=1&sid=...` 유효/무효/만료 케이스 처리 및 사용자 메시지 점검
- [ ] `sid` 조회 실패 시 `#s` fallback 복원 성공률 점검
- [ ] 공유 링크 오픈 후 클린 URL 복귀 동작(주소창 정리) 회귀 확인

### 백업
- [ ] 자동 백업 주기(12시간) 및 최대 보관(60개) 회전 정책 확인
- [ ] 수동 백업 레이트리밋(1분당 1개)과 덮어쓰기 확인 플로우 검증
- [ ] 백업 선택 복원 후 즉시 렌더/저장 상태 일치 확인

### PWA
- [ ] 오프라인 재진입 시 핵심 UI/데이터 접근 가능 여부 검증
- [ ] 새 버전 감지 토스트 + `registration.update()` 호출 경로 점검
- [ ] 버전 동기화 정책(`app.js`/`sw.js`/`manifest`) 릴리즈 체크리스트 운영
