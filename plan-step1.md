# Step1: 나의 가계 흐름 - 개발 계획

## 1. 현재 상태 요약
- **핵심 기능**: 수입/지출/저축/투자 항목 CRUD, Sankey Diagram 시각화, 10년 자산 추이 시뮬레이션.
- **저장/공유**: URL Hash 압축(`z:`), IndexedDB 공유 포인터(`sid`), 자동/수동 로컬 백업(IndexedDB).
- **PWA**: 서비스워커 캐싱, 오프라인 재진입, 버전 감지 및 업데이트 안내.

## 2. 기술 부채 및 UX 개선 후보
- **기술 부채**: `app.js`가 단일 파일로 비대해짐(약 3000라인). GitHub Pages 정적 호스팅(No-build) 환경을 고려해 브라우저 네이티브 ES Modules(`<script type="module">`) 방식의 분리 검토 필요.
- **UX 개선**: Sankey 노드 겹침 완화, 자유적립식 적금/파킹통장 같은 유동적 목돈 예적금 성격 입력 체계 보강 (TODO 보류 항목 연계).

## 3. 안정화 체크리스트
- [ ] 오프라인 상태에서 백업/복원 기능 정상 동작 확인
- [ ] `sid` 공유 링크 만료/오류 시 Hash Fallback 정상 전환 테스트
- [ ] PWA 백그라운드 업데이트 충돌 방지 (캐시 무효화 점검)

## 4. 우선순위 및 수용 기준(AC)
- **[Done] 샘플/초기화 데이터 유실 방지 (UX)**: 샘플은 View 모드, 초기화는 Confirm 후 금액만 리셋하도록 수정.
- **[Med] `app.js` 네이티브 모듈화 (refactor)**: 별도 빌드 도구 없이 `<script type="module">`을 활용해 UI/차트/상태 관리 등 도메인별로 파일 분리. (예상 난이도: 상)
- **[High] 보류된 목돈/파킹통장 입력 체계 구현 (feat)**: 고급 설정에 해당 탭 추가 및 시뮬레이션 정상 반영. (예상 난이도: 중)
- **[Med] Sankey Diagram 노드 겹침 방지 (UX)**: 극단적인 값 차이에도 라벨 및 노드가 겹치지 않음. (예상 난이도: 중)
- **[Low] 모바일 에러 피드백 강화 (UX)**: 잘못된 폼 입력 시 즉각적인 인라인 에러 메시지 노출. (예상 난이도: 하)
