# 통합 앱 간 데이터 브리지 및 자동 연동 패턴

## 개요
이 문서는 Step1(가계 흐름 시뮬레이션)의 결과 데이터를 Step2(투자 포트폴리오 설계)로 연동할 때, 사용자 조작 없이 즉시 반영(Sync by Default)하고 수동 수정 시에만 확인(Guard on Edit)하는 패턴을 정의합니다.

## 핵심 문제 및 원인 (AS-IS)
과거에는 Step 2 진입 시 사용자가 버튼을 눌러야만 데이터를 가져오거나, 값이 다를 때만 배너를 띄웠습니다. 이는 사용자에게 불필요한 선택을 강요하며 '데이터 재사용'의 흐름을 끊는 요소였습니다.

## 자동 연동 및 수정 가드 원칙 (TO-BE, v0.7.3)
에이전트는 다음의 사용자 경험(UX) 흐름을 보장하도록 로직을 설계해야 합니다.

1. 무조건적 자동 반영 (Sync by Default)
Step 2 로딩 시, `IsfStorageHub`에서 Step 1의 최신 데이터를 자동으로 조회하여 초안(draft)에 반영합니다. 사용자는 페이지 진입 즉시 Step 1 데이터가 반영된 시뮬레이션 그래프와 '월 투자 가능 금액(만원)'을 보게 됩니다.

2. 수정 시 의사 확인 (Guard on Edit)
자동 연동된 '월 투자 가능 금액' 필드를 사용자가 직접 수정하려고 하면, "연동된 데이터를 수정하시겠습니까? (수정 시 Step 1과의 실시간 연동이 해제될 수 있습니다)"와 같은 확인 메시지를 노출합니다.

3. 상태 보존 및 병합
데이터를 자동으로 반영할 때, 기존 Step 2에서 설정한 포트폴리오의 다른 값(수익률, 성장률 등)은 건드리지 않고 오직 '월 투자 여력'만 업데이트합니다.

4. 데이터 단위 정합성 (불변)
허브에서 수신되는 데이터는 원 단위이므로, UI 표시 시에는 반드시 `IsfUtils.toMan`을 거쳐야 합니다. (참고: [[Data_Model_Reference]])
